BiomedParse async AWS handoff
=============================

O handoff operacional atualizado está em:

`/home/antonio/medIA/development/vizier-inference-api/BIOMEDPARSE_ASYNC_AWS.md`

Resumo do desenho atual:

- sem EFS ou diretório compartilhado entre API e worker
- bucket S3 para `input.npz`, `output.npz` e `summary.json`
- DynamoDB para status de job
- SQS + DLQ para orquestração assíncrona
- worker ECS CPU despacha uma task ECS GPU por job
- task GPU usa storage efêmero local em `/tmp/jobs/<job_id>`
- contrato do runner:
  - `--input-file`
  - `--output-file`
  - `--device`
  - `--slice-batch-size`
  - `--summary-file`
